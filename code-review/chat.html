I want to advocate strongly for the rate limiting fix being P0. Here's why:

If someone deploys this to production with multiple instances (which is likely for a multi-tenant SaaS), the rate limiting is effectively disabled. An attacker could brute-force passwords by distributing requests across instances.

I'd say: Error Boundaries = P0 (affects everyone), Rate Limiting = P0 (security risk), Loading states = P1 (UX improvement)
